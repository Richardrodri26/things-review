# Internacionalización (i18n) — Things Review

> Estrategia de soporte multiidioma con **next-intl** para inglés y español.

---

## Idiomas soportados

| Código | Idioma | Estado |
|---|---|---|
| `en` | English | ✅ Principal |
| `es` | Español | ✅ Principal |

---

## Librería: next-intl

**next-intl** es la solución estándar para i18n en Next.js App Router. Soporta:
- Server Components y Client Components
- Rutas con locale en la URL
- Formateo de fechas y números según el locale
- Plurales, géneros gramaticales y mensajes con variables

### Instalación

```bash
bun add next-intl
```

---

## Estrategia de rutas

Las rutas tendrán el locale como prefijo:

```
/en/movies          → Catálogo en inglés
/es/movies          → Catálogo en español
/en/groups/abc123   → Detalle de grupo en inglés
/es/groups/abc123   → Detalle de grupo en español
```

El locale por defecto será `en`. Si el usuario navega a `/movies`, se redirige a `/en/movies`.

---

## Estructura de archivos de mensajes

```
messages/
  en.json    → Todos los textos en inglés
  es.json    → Todos los textos en español
```

## Configuración de next-intl

### `i18n/routing.ts`
```typescript
import { defineRouting } from 'next-intl/routing'

export const routing = defineRouting({
  locales: ['en', 'es'],
  defaultLocale: 'en',
})
```

### `i18n/request.ts`
```typescript
import { getRequestConfig } from 'next-intl/server'
import { routing } from './routing'

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale

  if (!locale || !routing.locales.includes(locale as 'en' | 'es')) {
    locale = routing.defaultLocale
  }

  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default
  }
})
```

## Uso en componentes

### Server Component
```typescript
import { getTranslations } from 'next-intl/server'

export default async function MoviesPage() {
  const t = await getTranslations('movies')

  return <h1>{t('title')}</h1>  // → "Movies" o "Películas"
}
```

### Client Component
```typescript
'use client'
import { useTranslations } from 'next-intl'

export function ReviewStatus({ status }: { status: WatchStatus }) {
  const t = useTranslations('reviews.status')

  return <Badge>{t(status)}</Badge>  // → "Watched" o "Vista"
}
```

### Mensajes con variables
```typescript
// "members": "{count, plural, one {# member} other {# members}}"
t('groups.members', { count: 5 })  // → "5 members" / "5 miembros"

// "averageRating": "Average: {rating}"
t('groups.averageRating', { rating: '4.2' })  // → "Average: 4.2"
```

## Detección de idioma del usuario

El idioma se detecta en este orden:

1. Locale en la URL (`/es/movies` → español)
2. Cookie guardada de preferencia anterior
3. Header `Accept-Language` del navegador
4. Default: `en`

---

## Consideraciones para el selector de idioma

Agregar un selector de idioma en el `AppHeader` que:
1. Muestre el idioma actual (`EN` / `ES`)
2. Al cambiar, navegue a la misma página con el nuevo locale
3. Guarde la preferencia en una cookie para visitas futuras

```typescript
import { useRouter, usePathname } from 'next/navigation'
import { useLocale } from 'next-intl'

export function LanguageSwitcher() {
  const locale = useLocale()
  const router = useRouter()
  const pathname = usePathname()

  const switchLocale = (newLocale: string) => {
    // next-intl provee el helper para esto
    router.push(pathname.replace(`/${locale}`, `/${newLocale}`))
  }

  return (
    <button onClick={() => switchLocale(locale === 'en' ? 'es' : 'en')}>
      {locale === 'en' ? 'ES' : 'EN'}
    </button>
  )
}
```
